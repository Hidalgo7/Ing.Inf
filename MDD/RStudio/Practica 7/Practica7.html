Construccin del modelo

Para empezar determinaremos una semilla inicial, en funcin de ella se crearn las particiones de train y test para el modelo.

set.seed("1234567890")

Una vez definida la semilla crearemos las particiones con el mtodo createDataPartition()

trainSetIndexes <- createDataPartition(y=iris$variety,p=.66,list=FALSE)

Es importante entender que parmetros recoge esta funcin y como cambia la salida en funcin de ellos

  • y \(\rightarrow\) variable que contiene los elementos a particionar.
  • p \(\rightarrow\) porcentaje de los casos que ira al subset de training.
  • list \(\rightarrow\) si especificamos FALSE para este parmetro devolver la particin en una matriz en lugar de en un vector.

Usando la variable en la que guardamos la particin y su negada como ndice para el dataset obtenemos las dos particiones.

trainSet <- iris[trainSetIndexes,]
testSet <- iris[-trainSetIndexes,]

Numero de elementos del subconjunto trainSet: 99

Numero de elementos del subconjunto testSet: 51

Guardaremos en la variable ctrl los parmetros que tendr nuestro clasificador, en nuestro caso un KNN.

ctrl <- trainControl("cv", number=10)
  • cv \(\rightarrow\) mtodo de clasificacion supervisada cross-validation
  • number \(\rightarrow\) valor para el hiperparametro del mtodo de clasificacin (en el caso del CV el nmero de hojas).

Por ltimo la funcin train() es la que genera el clasificador juntando todas las piezas que hemos generado hasta ahora.

KNNModel1 <- train(variety ~ ., data=trainSet, method="knn", tuneLength=5, trControl=ctrl, preProc=c("center","scale"))

La clase a predecir se marca en el primer parametro y separado por el caracter ~ se especifican las predictoras. Estos son los parmetros ms importantes que toma:

  • method \(\rightarrow\) especifica que metodo de clasificacin se va a usar para crear el clasificador.
  • tuneLength \(\rightarrow\) se usa para especificar para cuantos parmetros diferentes se probar el modelo. En nuestro caso se probarn 5 valores diferentes para K y el modelo se quedar con el que tenga mejor accuracy.
  • trControl \(\rightarrow\) trControl especifica que parametros queremos que tenga el clasificador, se usan los parametros ya especificados previamente.
  • preProc \(\rightarrow\) expresa el pre procesado previo que se le realiza a nuestros datos antes de entrenar el clasificador.
